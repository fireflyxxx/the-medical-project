![1777217266980](image/项目启动与路由说明/1777217266980.png)# 项目启动与路由说明

## 一、项目架构

本项目采用**双前端架构**：

| 端 | 技术栈 | 端口 | 说明 |
|---|---|---|---|
| 登录页/管理端/科研端 | Vite (React) | 80 (nginx 静态) / 5173 (开发) | 静态文件部署 |
| 医生端 | Next.js | 3000 | SSR 应用 |

**Nginx** 作为反向代理，根据路径分流到不同前端应用。

---

## 二、启动步骤

### 开发模式（推荐用于开发调试）

#### 1. 启动 Vite 开发服务器

```bash
cd d:/frontend-merge/frontend-vite
npm run dev
```

**特点：**
- 运行在 `http://localhost:5173`
- 实时热更新（HMR）
- 直接运行源代码，改动立即生效
- 适合开发阶段使用

**访问地址：**
- http://localhost:5173/login
- http://localhost:5173/admin
- http://localhost:5173/tech

**注意：** 开发模式下医生端跳转不可用，需要通过生产模式测试完整功能。

---

### 生产模式（完整功能测试）

#### 1. 构建 Vite 前端

```bash
cd d:/frontend-merge/frontend-vite
npm run build
```

**说明：** 
- 构建产物输出到 `frontend-vite/dist/` 目录
- 如遇 TypeScript 错误，需先修复后再构建
- 每次修改代码后都需要重新构建

#### 2. 启动 Next.js 医生端

```bash
cd d:/frontend-merge/frontend-node
npm start
```

> 确保服务运行在 **3000 端口**

#### 3. 启动 Nginx

```bash
cd d:/frontend-merge/nginx-1.26.3
nginx.exe
```

> 停止：`nginx.exe -s stop`  
> 重载配置：`nginx.exe -s reload`

#### 4. 访问应用

打开浏览器访问 **http://localhost/login**

## 三、开发与生产模式对比

| 模式 | 端口 | 服务方式 | 代码来源 | 实时更新 | 适用场景 |
|------|------|---------|---------|---------|---------|
| **开发模式** | 5173 | Vite Dev Server | `frontend-vite/src/` 源代码 | ✅ 是（HMR） | 日常开发、快速调试 |
| **生产模式** | 80 | Nginx 静态服务 | `frontend-vite/dist/` 构建产物 | ❌ 否（需重新构建） | 完整功能测试、部署前验证 |

**重要提示：**
- 开发模式下的改动只在 `localhost:5173` 生效
- 生产模式需要先执行 `npm run build` 构建，改动才会在 `localhost` 生效
- 如果改动只在 5173 端口有效，说明 dist 目录未更新，需要重新构建

## 四、URL 跳转逻辑

### 登录跳转（登录成功后）

| 登录角色 | 跳转地址 | 前端应用 |
|---|---|---|
| 医生（doctor） | `/doctor` | Next.js (frontend-node) |
| 管理员（admin） | `/admin` | Vite 静态页面 |
| 科研人员（tech） | `/tech` | Vite 静态页面 |

### 路由权限守卫

每个角色的页面都有权限保护，访问时会自动检查：

- **未登录**（无 token）→ 强制跳转到 `/login`
- **角色不匹配**（比如医生访问管理员页面）→ 自动跳转到自己对应的首页

### 退出登录

任意页面点击退出登录按钮 → 清除登录信息 → 跳转到 `/login`

### HTTP 401 自动登出

如果后端返回 401 未授权响应，前端会自动清除 token 并跳转到 `/login`。

## 六、Nginx 路由规则

```nginx
# 医生端：反向代理到 Next.js
location /doctor {
    proxy_pass http://127.0.0.1:3000;
}

# Next.js 静态资源
location /_next/ {
    proxy_pass http://127.0.0.1:3000;
}

# API 代理到后端
location /api/ {
    proxy_pass http://116.62.219.49:8080;
}

# 其他路径：Vite 静态文件
location / {
    root ../frontend-vite/dist;
}
```

## 七、常见问题排查

### 问题 1：改动只在 5173 端口有效，在 80 端口无效

**原因：** `frontend-vite/dist/` 目录是旧的构建产物，未包含最新改动。

**解决方案：**
```bash
cd d:/frontend-merge/frontend-vite
npm run build
cd ../nginx-1.26.3
nginx.exe -s reload
```

**重要提示：** 构建并重载 Nginx 后，如果浏览器仍显示旧版本，需要清除浏览器缓存：
1. 按 `Ctrl + Shift + Delete` 打开清除浏览器数据
2. 选择"缓存的图片和文件"
3. 点击清除数据
4. 刷新页面或按 `Ctrl + F5` 强制刷新

**快速方法：** 使用无痕模式/隐私模式打开浏览器测试，可避免缓存问题。

### 问题 2：构建失败，提示 TypeScript 错误

**原因：** 代码中存在类型错误。

**解决方案：**
1. 查看错误信息，定位到具体文件和行号
2. 修复类型错误
3. 重新执行 `npm run build`

**临时方案（不推荐）：** 修改 `package.json`，跳过类型检查
```json
"scripts": {
  "build": "vite build"  // 移除 tsc
}
```

### 问题 3：Nginx 启动失败或无法访问

**检查步骤：**
```bash
# 检查 80 端口是否被占用
netstat -ano | grep ":80"

# 查看 Nginx 错误日志
cd d:/frontend-merge/nginx-1.26.3/logs
cat error.log

# 停止 Nginx
nginx.exe -s stop

# 重新启动
nginx.exe
```

### 问题 4：医生端跳转失败（404）

**原因：** Next.js 服务未启动或端口不是 3000。

**解决方案：**
```bash
cd d:/frontend-merge/frontend-node
npm start
```

确认服务运行在 `http://localhost:3000`

## 九、后端地址配置（重要）

### 配置文件位置

后端服务器地址统一配置在 `frontend-vite/.env.local`：

```
VITE_API_BASE_URL=http://116.62.219.49:8080
```

### 更换服务器步骤

当需要更换后端服务器时，只需修改一个文件：

1. **修改配置文件** `frontend-vite/.env.local`：
   ```
   VITE_API_BASE_URL=http://新IP:新端口
   ```

2. **重新构建并部署**：
   ```bash
   cd d:/frontend-merge/frontend-vite
   npm run build:prod
   ```

**说明：**
- `npm run build` - 构建前端并自动生成 nginx 配置
- `npm run build:prod` - 构建并自动重载 nginx（一键部署）

### 配置原理

- **开发环境**：`vite.config.ts` 从 `.env.local` 读取后端地址
- **生产环境**：构建时自动生成 `nginx.conf`，从 `.env.local` 读取后端地址
- **配置模板**：`nginx-1.26.3/conf/nginx.conf.template`
