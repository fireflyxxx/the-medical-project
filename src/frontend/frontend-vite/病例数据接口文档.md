# 病例数据接口文档

## 后端服务地址

`http://116.62.219.49:8080`

---

## 主要接口

### 1. 获取病例列表

- **接口**: `GET /api/v1/cases/get`
- **请求头**: `Authorization: Bearer {token}`
- **查询参数**:
  - `case_id`（可选）：病例ID
  - `start_date`（可选）：起始日期
  - `end_date`（可选）：结束日期
- **返回**: `List<CaseDto>`

### 2. 获取所有推理结果

- **接口**: `GET /api/v1/infer_result/get`
- **请求头**: `Authorization: Bearer {token}`
- **返回**: `List<InferenceResultDto>`

### 3. 获取单个推理结果

- **接口**: `POST /api/v1/infer_result/get/{result_id}`
- **请求头**: `Authorization: Bearer {token}`
- **路径参数**: `result_id` - 推理结果ID
- **返回**: `InferenceResultDto`

### 4. 获取图片信息

- **接口**: `GET /api/v1/image/get/{image_id}`
- **请求头**: `Authorization: Bearer {token}`
- **路径参数**: `image_id` - 图像ID
- **返回**: `ImageDto`

## 数据结构

### 1. CaseDto - 病例基本信息

| 字段 | 类型 | 说明 |
|------|------|------|
| `caseId` | String | 病例ID |
| `name` | String | 患者姓名 |
| `gender` | Integer | 性别 |
| `age` | Integer | 年龄 |
| `idNumber` | String | 身份证号 |
| `contact` | String | 联系方式 |
| `medicalHistory` | String | 病史 |
| `createdTime` | LocalDateTime | 创建时间 |
| `updatedTime` | LocalDateTime | 更新时间 |
| `caseDesc` | String | 病例描述 |
| `studys` | List\<StudyDto\> | 检查列表 |

### 2. StudyDto - 检查信息

| 字段 | 类型 | 说明 |
|------|------|------|
| `studyId` | String | 检查ID |
| `studyTime` | LocalDate | 检查时间 |
| `studyType` | String | 检查类型 |
| `studyDesc` | String | 检查描述 |
| `imageIds` | List\<String\> | 图像ID列表 |

### 3. ImageDto - 图像信息

| 字段 | 类型 | 说明 |
|------|------|------|
| `imageId` | String | 图像ID |
| `caseId` | String | 病例ID |
| `studyId` | String | 检查ID |
| `fileName` | String | 文件名 |
| `fileFormat` | String | 文件格式 |
| `fileSize` | Long | 文件大小 |
| `imagePath` | String | 图像路径 |
| `uploadedTime` | LocalDateTime | 上传时间 |

### 4. InferenceResultDto - AI推理结果

| 字段 | 类型 | 说明 | 实际情况 |
|------|------|------|----------|
| `resultId` | String | 结果ID | ✅ 正常返回 |
| `imageId` | String | 图像ID | ⚠️ 可能为 null |
| `caseId` | String | 病例ID | ⚠️ 可能为 null |
| `originalImgPath` | String | 原始图像路径 | ⚠️ 可能为 null |
| `bbox` | Object | 标注框（可以是单个Map或List\<Map\>） | ✅ 正常返回 |
| `confidenceScore` | Double | 置信度 | ✅ 正常返回 |
| `label` | String | 标签 | ✅ 正常返回 |
| `createdTime` | LocalDateTime | AI推理完成时间 | ✅ 正常返回 |
| `isModified` | Boolean | 是否被修改 | ✅ 正常返回 |
| `modifiedBbox` | Object | 修改后的标注框 | ✅ 正常返回（可能为空数组 []） |
| `modifiedLabel` | String | 修改后的标签 | ✅ 正常返回 |
| `modifiedBy` | String | 修改人 | ✅ 正常返回 |
| `modifiedTime` | LocalDateTime | 修改时间 | ✅ 正常返回 |
| `annotatedImgPath` | String | 标注图像路径 | ✅ 正常返回 |
| `comment` | CommentDto | 评论 | ✅ 正常返回 |

**重要说明**:
- `imageId` 和 `caseId` 在某些推理结果中可能为 `null`，这是数据库中的数据问题
- `modifiedBbox` 为空数组 `[]` 表示医生删除了所有标注框
- `originalImgPath` 可能为 `null`，但通常 `annotatedImgPath` 有值

### 5. CommentDto - 评论信息

| 字段 | 类型 | 说明 |
|------|------|------|
| `commentId` | String | 评论ID |
| `satisfaction` | String | 满意度 |
| `sentence` | String | 评论内容 |

## 数据层级关系

```
Case (病例) - caseId
  └─ Study (检查) - studyId
       └─ Image (图像) - imageId
            └─ InferenceResult (AI推理结果) - resultId
                 └─ Comment (评论) - commentId
```

## 已知问题

### 1. 推理结果中的 imageId 和 caseId 为 null

**问题描述**: 部分推理结果的 `imageId` 和 `caseId` 字段在数据库中为 `null`

**影响**: 
- 前端无法直接显示样本编号和病例ID
- 需要通过其他接口补充这些信息

**前端解决方案**:
1. 通过病例列表接口获取 `caseId`（从 Case → Study → imageIds 的关联）
2. 通过图片信息接口补充 `caseId`
3. 缺失的数据显示为 "-"

### 2. modifiedBbox 为空数组

**问题描述**: 当医生删除所有标注框时，`modifiedBbox` 返回空数组 `[]`

**前端处理**: 
- 检查 `modifiedBbox` 是否为空数组
- 如果为空，显示原始的 `bbox`
- 如果原始 `bbox` 也没有，显示提示信息
