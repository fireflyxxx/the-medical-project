# UI/UX 设计规范
## 医疗影像AI分析平台

---

## 1. 设计原则

### 1.1 专业可信 (Professional & Trustworthy)
医疗场景要求界面传达专业性和可靠性，避免过度装饰和娱乐化元素。

### 1.2 清晰高效 (Clear & Efficient)
医生工作节奏快，界面需要信息层级清晰，操作路径简短，减少认知负担。

### 1.3 安全合规 (Secure & Compliant)
敏感数据脱敏展示，关键操作二次确认，操作可追溯。

### 1.4 包容友好 (Inclusive & Friendly)
考虑不同年龄段医生的使用习惯，提供清晰的反馈和引导。

## 6. 交互规范

```
全屏加载：
- 遮罩：rgba(255, 255, 255, 0.9)
- 加载图标：旋转的医疗十字图标
```

#### 局部加载
```
骨架屏：
- 用于列表、卡片加载
- 灰色占位块 + 闪烁动画
```

#### 按钮加载
```
按钮内图标旋转
文字变为"处理中..."
禁用点击
```

### 6.2 反馈机制

#### 成功反馈
```
消息提示：顶部居中，3秒自动消失
图标：✓ 绿色对勾
文字："操作成功"
```

#### 错误反馈
```
消息提示：顶部居中，5秒自动消失
图标：✗ 红色叉号
文字：具体错误信息
```

#### 警告反馈
```
消息提示：顶部居中，4秒自动消失
图标：⚠ 橙色感叹号
文字：警告信息
```

### 6.3 确认机制

#### 二次确认弹窗
```
标题：明确操作名称（如"删除病例"）
内容：
- 警示图标（红色）
- 警示文字："此操作将永久删除该病例及关联的所有影像和推理记录，是否继续？"
- 重要信息高亮显示

按钮：
- 取消（次要按钮，左侧）
- 确认（危险按钮，右侧）
```

### 6.4 表单验证

#### 实时验证
```
失焦验证：输入框失去焦点时验证
错误提示：输入框下方显示红色错误文字
成功提示：输入框右侧显示绿色对勾图标
```

#### 提交验证
```
验证失败：
- 滚动到第一个错误字段
- 错误字段高亮
- 顶部显示错误汇总消息
```

### 6.5 空状态

#### 空列表
```
图标：灰色文件夹图标
文字："暂无数据"
操作：引导按钮（如"创建第一个病例"）
```

#### 搜索无结果
```
图标：灰色搜索图标
文字："未找到相关结果"
建议："请尝试其他搜索条件"
```

## 9. 可访问性 (Accessibility)

### 9.1 键盘导航
- 所有交互元素支持 Tab 键导航
- 焦点状态清晰可见（蓝色外框）
- 支持 Enter/Space 触发操作

### 9.2 屏幕阅读器
- 图标添加 aria-label
- 表单添加 label 关联
- 动态内容添加 aria-live

### 9.3 对比度
- 文字与背景对比度 ≥ 4.5:1
- 大文字（≥18px）对比度 ≥ 3:1
