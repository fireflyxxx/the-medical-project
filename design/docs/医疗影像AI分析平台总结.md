# 医疗影像AI分析平台总结

## 项目概述

基于AI驱动的医疗影像分析平台，用于辅助医生进行肺炎等疾病的影像诊断。系统采用RBAC权限控制，支持DICOM医学影像格式，实现从数据上传到AI推理再到结果导出的完整闭环。

## 用户角色

### 系统管理员 (System Administrator)
- 负责用户管理、模型版本配置及参数设置
- 创建/禁用账号、查看运行日志、审计敏感操作

### 科研人员 (Researcher)
- 查看医生提交的案例与AI识别结果
- 根据反馈调整AI模型配置和参数

### 超声科医生 (Sonographer)
- 平台主要使用者
- 上传影像、发起AI检测、查看和评价结果

## 核心工作流程（9个标准动作）

### 阶段一：数据准备 (Data Preparation)
1. **用户登录** - 身份验证，触发RBAC权限控制
2. **创建/选择数据** - 新建病例或检索已有记录
3. **上传影像数据** - 支持DCM/PNG/JPG格式，自动脱敏处理

### 阶段二：核心推理 (Core Inference)
4. **选择模型与参数** - 从模型库选择AI模型并配置参数
5. **提交推理任务** - 前端发出指令，任务加入队列
6. **后端调用AI模型** - 服务器执行特征提取与病灶识别

### 阶段三：结果交付 (Result Delivery)
7. **保存结果与日志** - 持久化存储推理数据和操作日志
8. **前端可视化展示** - 叠加检测框/掩码，支持交互操作
9. **导出结果** - 输出JSON数据和标注截图/报告

## MVP功能需求（7项核心功能）

| 编号 | 功能模块 | 核心要点 |
|------|----------|----------|
| FR-01 | 账号与权限管理 | RBAC机制，区分三类角色权限 |
| FR-02 | 影像数据管理 | 病例检索、影像上传、脱敏处理 |
| FR-03 | 本地AI模型集成 | 加载至少1个本地模型，提供统一接口 |
| FR-04 | 推理任务与存储 | 保存任务状态、耗时及推理结果 |
| FR-05 | 结果可视化展示 | 检测框/掩码展示，支持阈值过滤 |
| FR-06 | 结果导出 | 支持JSON + 标注图导出 |
| FR-07 | 日志与审计基础 | 记录关键操作，支持审计查询 |

## 核心特性

- **严谨性** - 先保存日志/结果再可视化，确保数据安全
- **灵活性** - 支持模型参数动态选择和微调
- **合规性** - 全流程RBAC控制，满足医疗审计要求
- **专业性** - 支持DICOM协议和影像脱敏
- **闭环性** - 从运维到临床使用的全覆盖

## 技术要点

- 支持DICOM医学影像标准格式
- 基于RBAC的权限控制机制
- 本地AI模型集成与推理
- 完整的操作日志与审计追溯
- 前端可视化标注与交互
- 结构化数据导出能力
