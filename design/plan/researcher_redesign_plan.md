# 科研人员端界面重构计划

## 项目背景

当前科研人员端界面存在功能定位不清晰的问题。经过需求澄清,明确了系统定位:
- **这是一个医疗影像AI应用平台**,不是AI模型训练平台
- 科研人员在本地环境训练模型,通过系统获取数据和上传模型
- 系统主要用于数据管理、模型部署和性能监控

## 核心功能定位

### 科研人员的实际工作流程
```
1. 从系统下载医生标注的影像数据
2. 在本地环境(GPU服务器/云平台)训练模型
3. 训练完成后,上传新模型到系统
4. 监控模型在生产环境的表现
5. 根据反馈数据优化模型
```

## 界面调整方案

### 一、保留并优化的功能

#### 1. 可视化展示 (第93-157行)
**当前问题:**
- 显示的是训练过程指标(Loss、Accuracy、Epoch进度)
- 给人感觉是在系统内训练模型

**调整方案:**
- 改为显示**生产环境模型性能监控**
- 指标调整:
  - ~~训练损失 (Loss)~~ → **模型准确率 (Accuracy)**
  - ~~精确度 (Accuracy)~~ → **医生同意率**
  - ~~召回率 (Recall)~~ → **误诊率**
  - ~~数据集大小~~ → **累计诊断案例数**
- 图表调整:
  - ~~模型收敛曲线~~ → **模型性能趋势图**(按时间显示准确率变化)
  - ~~训练样本类别占比~~ → **医生反馈分布**(同意/需复核/不同意)
- 删除"Epoch 85/100"等训练进度信息

#### 2. 影像数据审查 (第160-384行)
**当前状态:** ✅ 功能完整,保持不变
- 数据表格展示
- 医生评价信息
- 快速统计卡片
- 导出报告功能

#### 3. AI模型配置 (第387-594行)
**当前问题:**
- 界面暗示可以在系统内启动训练
- 说明文字容易误导用户

**调整方案:**
- 页面标题: "AI模型参数配置" → **"模型训练参数记录"**
- 副标题: "调整训练超参数..." → **"查看和管理模型训练参数配置(仅供参考,实际训练请在本地环境进行)"**
- 按钮调整:
  - ~~"保存配置"~~ → **"导出配置文件"** (下载JSON/YAML)
  - 新增: **"查看历史配置"** 按钮
- 底部说明文字调整:
  - 原文: "配置将在下次训练任务启动时生效..."
  - 改为: **"此配置为当前部署模型的训练参数记录。您可以导出此配置用于本地训练参考,或上传新模型时填写对应的训练参数。"**

## 设计原则

1. **明确系统定位** - 所有文案和功能都要体现"应用平台"而非"训练平台"
2. **简化操作流程** - 核心操作(数据导出、模型上传)不超过3步
3. **保持视觉一致性** - 沿用现有的紫色主题和卡片式设计
4. **提供清晰反馈** - 每个操作都有明确的成功/失败提示
5. **支持批量操作** - 数据导出支持批量选择,模型管理支持批量对比

## 风险与注意事项

1. **数据安全** - 数据导出需要权限控制和审计日志
2. **模型验证** - 上传的模型需要格式验证和基础测试
3. **版本兼容** - 确保新旧模型版本的接口兼容性
4. **性能监控** - 生产环境指标需要实时更新机制
5. **用户培训** - 功能调整后需要更新使用文档
