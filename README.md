# SE3351 软件开发实践 - 课程项目模板

本仓库为 **软件开发实践** 课程的项目模板，供学生小组克隆使用。请在此模板基础上补充完善需求文档、设计文档、测试文档、项目代码等内容，最终将以各小组的 **master 分支** 作为课程考核的提交内容。

## 使用流程

1. **获取仓库**：通过 GitHub Classroom 接受作业后，你将获得基于本模板创建的个人/小组仓库
2. **克隆到本地**：`git clone <你的仓库地址>`
3. **补充完善**：按课程要求撰写文档、编写代码、执行测试
4. **提交考核**：课程结束时，教师将以你仓库的 **master 分支** 为准进行评分

> ⚠️ **重要提醒**：请确保所有需要评分的文档和代码都已提交到 master 分支，并完成 `git push`。

## 目录结构说明

```
SE3351/
├── README.md           # 本说明文件
├── 模板/               # 各类文档模板（请参考并补充到 docs/ 或相应目录）
│   ├── 1-需求模板/     # 软件需求规格说明书等
│   ├── 2-设计模板/     # 软件设计说明书等
│   ├── 3-测试模板/     # 测试计划、测试用例、测试报告等
│   └── 4-项目管理相关模板/  # 项目计划、周报、结项申请等
├── docs/               # [建议] 将完善后的各类文档放于此目录
└── src/                # [建议] 项目源代码目录
```

## 提交物清单

请根据课程要求完成并提交以下内容（具体以授课教师为准）：

| 类型 | 说明 |
|------|------|
| 需求文档 | 软件需求规格说明书（SRS） |
| 设计文档 | 软件设计说明书 |
| 测试文档 | 功能/接口测试计划、测试用例、测试报告 |
| 项目管理 | 项目计划、周报、结项申请 |
| 源代码 | 项目可运行代码 |

## 文档命名规范

可参考 `模板/` 目录下文件命名规则，例如：

- 需求：`Gxx_XXX系统_软件需求规格说明书_vXX.docx`
- 设计：`Gxx_xx系统_软件设计说明书_vXX.docx`
- 测试：`GXX_XX系统_功能测试计划_v0.1.docx` 等

## 常见问题

**Q：如何知道自己是否已正确接受作业？**  
A：在 GitHub Classroom 接受作业后，会跳转到你的专属仓库页面，该仓库即为你需要使用的项目仓库。

**Q：master 分支和 main 分支有什么区别？**  
A：本课程统一以 **master** 分支为考核依据，请确保所有内容已提交到 master 分支。

**Q：可以创建其他分支吗？**  
A：可以，用于日常开发，但最终需将考核内容合并到 master 分支。
